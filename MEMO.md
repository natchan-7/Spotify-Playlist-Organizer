## 実装メモ

### アーティスト情報の使い道

- `genres`: 自動タグ、集計、フィルターに使える
- `id`: キャッシュキー、再取得、絞り込み条件に使える
- `name`: 表示、検索、補助タグに使える
- `images`: アーティスト詳細 UI に使える
- `external_urls.spotify`: Spotify への外部リンクに使える
- `popularity`: 並び替えや分析表示に使える
- `followers.total`: 補助情報として表示できる

### API まわり

- プレイリスト内アイテム取得は `GET /playlists/{id}/items` を基準にする
- アーティスト情報取得は基本的に `GET /artists?ids=...` の一括取得を使う
- 単体取得 `GET /artists/{id}` は手入力検索や詳細画面で使う余地がある
- `genres` が空のアーティストもあるので、空配列を前提に扱う
- OpenAPI schema をエンドポイントの正本として扱う
- `429` は `Retry-After` を優先し、なければ指数バックオフする

### UI まわり

- 小さく始めるなら「リンク化」「集計表示」「一覧表示」から入る
- クリック操作を増やす場合は、曲一覧の可読性を落とさないようにする
- 新しい情報を足すときは、まず折りたたみか補助カードで検証する

### 準拠ルール

- 認証は Authorization Code with PKCE を使う
- redirect URI は HTTPS を使い、ローカル開発時のみ `http://127.0.0.1` を使う
- `http://localhost` は使わない
- Client Secret はフロントエンドに置かない
- Spotify 由来データは長期キャッシュしない
